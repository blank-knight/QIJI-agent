# 知识库与 GEO 联动机制

## 自动化流程

### 场景1：用户日常对话 → 知识库积累

```
用户: "我们公司是做智能手表的，品牌叫星表，主打健康监测..."
Agent:
  1. 识别到品牌信息（星表、智能手表、健康监测）
  2. 读取 ~/.hermes/knowledge-base/brands/星表.md（如果存在）
  3. 追加新信息（不覆盖已有内容）
  4. 更新 _index.md
  5. 正常回复用户，末尾加 "💡 已更新知识库：星表"
```

### 场景2：GEO 诊断 → 自动填充

```
用户: "帮我诊断一下星表在AI搜索里的表现"
Agent:
  1. 加载 qiji-geo skill
  2. 加载 qiji-knowledge-base skill
  3. 读取 brands/星表.md → 获取品牌名、关键词
  4. 读取 industry/智能穿戴.md → 补充行业关键词
  5. 自动填入 geo-cli.js diagnose --brand 星表 --keywords "智能手表,健康监测,..."
  6. 不需要用户再手动报品牌和关键词
```

### 场景3：诊断结果 → 反哺知识库

```
GEO 诊断完成后:
  1. 诊断报告中的关键词表现数据 → 追加到 brands/星表.md 的 GEO 关键词章节
  2. 表现好的关键词标记为"AI搜索友好"
  3. 表现差的标记为"需优化"
  4. 下次 AI 写作时优先使用表现好的关键词
```

### 场景4：AI 写作 → 基于知识库生成

```
用户: "帮我写一篇关于星表的文章"
Agent:
  1. 读取 brands/星表.md → 品牌调性、卖点
  2. 读取 products/ → 产品详情
  3. 读取 content/ → 参考优质内容风格
  4. 读取 brands/星表.md 的 GEO 关键词 → 嵌入关键词
  5. 生成基于真实资料的文章
```
