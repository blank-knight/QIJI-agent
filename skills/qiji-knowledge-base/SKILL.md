# 奇计知识库

自动收集和管理用户的知识资产。用户日常对话中提到的品牌信息、产品资料、行业知识，
会被自动整理成结构化的知识库文件。GEO 诊断、AI 写作、爆文复刻时自动调用这些知识。

## 核心理念

用户不需要刻意"建知识库"——他们在正常对话中提到的一切，都会被自动归类存储。
当需要做 GEO 诊断或内容创作时，知识库已经有足够的素材了。

## 知识库路径

```
~/.hermes/knowledge-base/
├── brands/          # 品牌资料
├── products/        # 产品信息
├── industry/        # 行业知识
├── customers/       # 客户/目标用户画像
├── competitors/     # 竞品分析
├── content/         # 优质内容存档（爆文、案例）
└── _index.md        # 知识库索引（自动维护）
```

## 自动收集触发词

当用户说以下内容时，自动提取并存入知识库：

| 用户说的话 | 提取内容 | 存入目录 |
|-----------|---------|---------|
| "我们公司是做..." | 公司简介、主营业务 | `brands/{品牌名}.md` |
| "我们的产品..." | 产品名、卖点、定价 | `products/{产品名}.md` |
| "我们的客户主要是..." | 目标用户画像 | `customers/{客户类型}.md` |
| "竞品XXX..." | 竞品信息、优劣势 | `competitors/{竞品名}.md` |
| "这个行业..." | 行业趋势、关键词 | `industry/{行业名}.md` |
| "这篇文章不错/存一下" | 爆文存档 | `content/{日期}-{标题}.md` |
| "记住这个" / "存一下" | 上下文中的关键信息 | 按类型自动归类 |

## 操作流程

### 1. 自动收集（被动模式）

用户正常聊天 → Agent 识别到品牌/产品/行业信息 → 静默提取 → 追加到对应文件
→ 更新 `_index.md` → 在回复末尾轻提示"已存入知识库"

**关键原则：不打断对话。** 收集是后台行为，不让用户觉得被问卷轰炸。

### 2. 主动检索（GEO 诊断时）

当 qiji-geo skill 需要品牌信息时：
1. 读取 `brands/{品牌名}.md` 获取品牌资料
2. 读取 `products/` 下该品牌的产品列表
3. 读取 `industry/` 获取行业关键词
4. 自动填充 GEO 诊断表单的品牌名、关键词等字段

### 3. 内容创作（AI写作时）

当用户要求写文章/文案时：
1. 检索知识库中相关品牌和产品资料
2. 检索 `content/` 中的历史优质内容作为风格参考
3. 基于真实资料生成内容（杜绝编造）

### 4. 手动管理

用户可以直接对话管理知识库：
- "看看我的知识库" → 显示 `_index.md` 内容
- "更新华为的品牌资料" → 打开文件，用户口述更新
- "删掉过时的产品信息" → 确认后删除
- "搜一下知识库里关于手机的内容" → 全文搜索

## 文件格式

每个知识条目用统一格式（模板见 `templates/brand-template.md`）：

```markdown
# {名称}
> 最后更新: YYYY-MM-DD | 来源: 用户对话/手动录入

## 基本信息
- 类型: 品牌/产品/客户/竞品/行业
- 标签: [标签1, 标签2]

## 详细内容
{用户提供的所有信息}

## 关联
- 相关品牌: [[brands/xxx]]
- 相关产品: [[products/xxx]]
```

## 与 GEO Skill 联动

知识库是 GEO 优化的弹药库：

```
用户日常对话 → 知识库积累
                    ↓
        GEO诊断时自动填充品牌/关键词
                    ↓
        诊断结果反馈 → 更新知识库（哪些关键词表现好）
                    ↓
        AI写作时调用知识库素材 → 生成优质内容
                    ↓
        发布效果好的内容存入 content/ → 下次参考
```

## 批量导入（用户有现成资料时）

### 支持的文件格式

| 格式 | 处理方式 | 示例 |
|------|---------|------|
| PDF (.pdf) | pymupdf 提取文本 | 公司画册、产品手册、行业报告 |
| Word (.docx) | 自动提取文本 | 产品介绍、品牌故事、合同模板 |
| Excel (.xlsx/.csv) | 表格→结构化数据 | 产品价格表、客户清单、竞品对比 |
| 文本 (.txt/.md) | 直接读取 | 笔记、文案草稿、会议记录 |
| 图片 (.png/.jpg) | vision_analyze OCR | 截图、海报、名片、产品图 |

### 导入流程

**第一步：扫描文件**

```bash
python3 scripts/import_data.py "C:\Users\用户\桌面\我的资料" --json
```

返回文件清单（JSON），包含每个文件的类型、大小、推荐分类。

**第二步：逐个处理（Agent 自动完成）**

对每个文件，Agent 执行：
1. 用 `read_file` 读取文件内容（docx/xlsx/pdf 自动提取）
2. 分析内容，判断类型（品牌/产品/客户/竞品/行业/内容存档）
3. 提取关键信息，填入对应模板
4. 写入知识库对应目录
5. 更新索引

**第三步：确认和补充**

Agent 处理完后，列出导入结果摘要，让用户确认或补充。

### 用户操作方式

用户可以用对话触发批量导入：

- "把这些文件导入知识库" + 提供文件夹路径
- "导入这个 PDF" + 提供文件路径
- "把桌面上的资料都导入知识库"
- "把这个 Excel 里的产品导入知识库"

### 导入后处理

每个文件导入后，Agent 自动：
1. 按模板格式整理成 markdown
2. 放入对应的分类目录
3. 提取关键词标签
4. 更新 `_index.md` 索引
5. 报告导入结果（成功/失败/需人工确认）

### 批量导入规则

1. **文件名智能分类**：文件名含"品牌"→brands，含"产品"→products，含"竞品"→competitors 等
2. **内容二次判断**：文件名不明显时，读内容判断分类
3. **表格特殊处理**：Excel/CSV 每行可能是一个独立条目（如产品列表），拆分存储
4. **去重**：同名文件跳过，内容相似的追加而非新建
5. **大文件分段**：超过 100K 字符的文件分段读取
6. **错误处理**：解析失败的文件跳过，不影响其他文件

## Agent 使用规则

1. **用户提到品牌/产品/公司信息** → 提取关键信息 → 存入对应文件 → 更新索引
2. **用户说"记住/存一下"** → 分析上下文 → 按类型归类存储
3. **用户说"导入"** → 运行 import_data.py 扫描 → 逐个读取处理 → 分类存储
4. **GEO诊断前** → 自动检索知识库 → 预填品牌和关键词
5. **AI写作前** → 自动检索知识库 → 基于真实资料写作
6. **用户问"知识库里有什么"** → 显示索引 → 高亮最近更新
7. **用户说"搜知识库"** → 全文搜索 → 返回匹配条目

## 注意事项

- **绝不编造知识库内容**：只存用户明确说过或确认的信息
- **去重**：同一品牌的新信息追加到已有文件，不创建重复文件
- **用户可编辑**：知识库文件是普通 markdown，用户可以随时直接编辑
- **隐私**：知识库存在本地 `~/.hermes/knowledge-base/`，不上传任何地方
- **增量更新**：每次更新只追加/修改相关部分，不重写整个文件
