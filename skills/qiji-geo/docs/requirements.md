## 1. 背景

### 1.1 什么是奇计GEO

奇计GEO是一个**企业AI可见度优化平台**，帮助企业在各大AI搜索引擎（DeepSeek、豆包、Kimi、智谱、元宝等）中提升品牌和产品曝光。平台分两部分：

- **网页端** (geo.heikexia.cc) — 诊断、知识库、指令、关键词、文章管理、SEO发布
- **桌面客户端** (auth helper.exe) — 16个社媒平台分发 + 8个AI认证

### 1.2 本 Skill 的定位

本 Skill 随奇计离线安装包分发给客户，让 Hermes Agent 能自动操作GEO平台。客户只需对话即可完成GEO全流程，不需要手动点网页。

### 1.3 平台技术架构

```
客户电脑
├── 奇计桌面版 (Qiji.exe)
│   └── Hermes Agent
│       └── qiji-geo skill ← 本文档描述的东西
│           ├── geo-cli.js   ← 网页端自动化 (Playwright)
│           └── geo-client.py ← 桌面客户端自动化 (Flask API)
│
├── GEO 网页端 (geo.heikexia.cc)
│   └── Bootstrap Table + iframe 架构
│
└── auth helper.exe (桌面客户端)
    ├── 本地 Flask API (127.0.0.1:5000) ← 浏览器自动化
    └── 远程服务器 (8.138.58.181) ← 管理类API
```

## 3. 需求规格

### 3.1 三大使用场景

```
场景一：写、发（企业GEO）
  打开GEO客户端登录 → 写作文章 → 发布到12个自媒体平台

场景二：查（收录查询）
  用客户端查询企业GEO收录情况

场景三：复盘、优化（半月一次）
  ① 重走AI可见度诊断 → 查看诊断报告
  ② 客户端查收录 → 查看数据报表
```

### 3.2 GEO全流程

```
客户首次使用
│
├─ 1. 知识库建设 ← 需求缺口
│   ├─ 客户提供真实资料（公司介绍、产品资料、图片等）
│   ├─ skill 引导客户构建知识库
│   └─ 导入到奇计网页端（企业画像图库 + 企业知识库）
│
├─ 2. 指令配置 ← 需求缺口
│   ├─ skill 根据默认模板引导客户写12个平台的指令
│   ├─ 每个平台3类指令：文章创作 / 标题创作 / 流量复刻
│   └─ 导入到奇计网页端
│
├─ 3. AI可见度诊断 ← 已实现（diagnose命令）
│   ├─ 填写品牌名 + 行业关键词
│   ├─ 选择AI平台（DeepSeek/豆包/Kimi/智谱/元宝等）
│   └─ 提交诊断（13元/次）
│
├─ 4. 文章写作 ← 已实现（write-tasks命令可查，写入待开发）
│   ├─ 根据知识库 + 诊断结果 + 指令，AI自动写作文章
│   ├─ TODO: AI自己写 vs 爆文复刻的选择机制（需求未定）
│   └─ 文章进入待审核状态
│
├─ 5. 文章审核 ← 需求缺口（强约束：人工审核）
│   ├─ ⚠️ 文章待审核必须用户手动通过（担责问题）
│   └─ skill 提醒用户审核，不自动操作
│
├─ 6. 文章发布 ← 部分实现
│   ├─ 6a. 自媒体发布：网页端创建发布任务 → 客户端执行
│   │   ├─ 网页端：选文章分类、选平台、设发布量
│   │   ├─ 客户端：检测到任务后自动发布
│   │   └─ 支持8个平台：网易/搜狐/百家号/头条号/企鹅号/知乎/公众号/小红书
│   │
│   └─ 6b. AI官网SEO发布：站点管理 → 栏目映射 → SEO发布任务
│       ├─ 分组名必须对应官网后台栏目名
│       ├─ 首次使用或改栏目名时，需截图喂给AI记住栏目↔ID映射
│       └─ 后台默认账号：admin / 15108232176
│
├─ 7. GEO优化引导 ← 需求缺口
│   ├─ 让用户选择优先优化哪个国家 / 省市区
│   ├─ 逐级推进：四川→成都→各区县 的关键词文章写发
│   └─ 然后推进其他省市区
│
└─ 8. 收录查询 + 复盘 ← 已实现（含澄清）
    ├─ 半月一次：重跑AI诊断 → 看诊断报告
    │   ⚠️ "收录查询"= 诊断报告本身，不是独立功能
    │   远程API(8.138.58.181)无收录查询端点
    │   /api/zhushou/get_tongji 只有发布统计（账号数/今日发布/累计发布/文章数）
    └─ dashboard + consumption 命令覆盖基础数据查询
```

> **⚠️ 收录查询澄清（2026-07-05 逆向验证）**
> 幕布原文"客户端查收录"= 企业在各AI搜索引擎中的被收录/引用情况。
> 这个数据来源就是**AI可见度诊断报告**——诊断跑完，报告里就是各平台的收录数据。
> 远程API和本地Flask均无独立的"收录查询"端点。
> 因此"查收录"不是独立功能，`report` 命令就是收录查询的结果展示。

## 4. 需求缺口详细分解

### 4.1 知识库引导流程【P0】

**需求描述：**
客户首次使用时，skill 引导客户提供真实资料，构建知识库后导入网页端。

**具体要求：**
- 引导客户提供：公司介绍、产品/服务资料、品牌方向（默认全选：经验分享/过程分享/用户痛点/用户避坑/科技前沿/行业洞察/潮流趋势等）
- 图片不好弄就让客户自己上传
- 网页端知识库分两个：**企业画像图库** + **企业知识库**（图1红框标注）
- 知识库预览要求：用通俗易懂语言描述，避免专业术语（图3）

**实现方案：**
- skill 触发时引导客户提供资料（对话式）
- 资料整理成结构化格式
- 提示客户手动导入到网页端（或后续开发自动导入）
- 与 `knowledge-base-articles` skill 联动读取品牌数据

**强约束：** 知识库必须基于真实资料，不能编造。

---

### 4.2 指令配置引导【P0】

**需求描述：**
12个平台必须有各自独立的指令，每个平台3类指令。

**具体要求：**
- 三个标签页：文章创作 / 标题创作 / 流量复刻（图4）
- skill 根据默认指令模板引导客户写自己的指令
- 示例指令格式（标题类）：
  - 25-30字
  - 加入年份2026
  - 引起好奇心但不夸张
  - 准确反映核心内容
  - 有节奏感
- 对应AI写的文章应发布到对应平台（元宝→微信公众号之类）

**实现方案：**
- skill 内置12平台的默认指令模板（参考压缩包文档）
- 触发时让客户选择平台 → 生成定制指令建议
- 客户确认后提示导入网页端

**强约束：** 每个平台每类指令都必须有，缺一个该平台就不能正常工作。

---

### 4.3 SEO网站发布【P1】

**需求描述：**
通过SEO发布任务，将文章发布到客户官网的对应栏目。

**具体要求：**
- 前提：站点管理里有客户官网域名（图12）
- 栏目映射：分组名对应官网后台栏目名（图13-14）
- 栏目列表：栏目名称/编码/URL名称/模型/列表页模板/详情页模板/排序/状态（图5、14、16）
- 典型栏目：公司简介(aboutus) / 新闻中心(article) / 产品中心(product) / 服务案例(case) / 招聘信息(job) / 联系我们(contact)
- 首次使用或改栏目名时，需手动给AI后台截图，让AI记住栏目↔ID映射（图5、16）
- SEO发布任务：任务名 / 站点 / 发布栏目 / 文章分类 / 发布日期范围 / 每日发布量（图15）
- 后台账号：admin / 15108232176

**实现方案：**
- 新增 `seo-sites` 命令：查看站点列表
- 新增 `seo-columns` 命令：查看栏目列表（栏目名↔ID映射）
- 新增 `seo-publish` 命令：创建SEO发布任务
- 截图识别功能：客户提供后台截图 → 识别栏目→ 存储映射关系

### 4.4 拓词工具【待定/低优先级】

> ⚠️ 幕布原文标注为 TODO List，优先级低。AI自动拓词（生成新词）需求也未定。

**需求描述：**
AI辅助拓词，通过6类词库组合生成大量长尾关键词。

**6类词库（图21）：**

| 类别 | 标签 | 示例 |
|------|------|------|
| A 前缀1 | 市/行业 | 市面上、行业内、市场、目前、国内 |
| B 前缀2 | 修饰 | 口碑好的、正规的、专业的、热门的、知名的 |
| C 主词 | 核心 | 洗地机、机器人、注塑机（客户产品） |
| D 通义词 | 行业 | 品牌、公司、工厂、厂家、供应商、制造商 |
| E 推荐词 | 口碑 | 名声、口碑 |
| F 疑问词 | 需求 | 哪家好、哪家强、哪家靠谱、哪个好、有哪些 |

**组合方式：**
C+D / A+C+D / B+C+D / A+B+C+D / C+D+E / C+D+F / A+C+D+E / B+C+D+E / A+B+C+D+E / A+B+C+D+F

**实现方案：**
- skill 内置6类词库默认值
- 客户提供主词（C）后，自动组合生成关键词列表
- 结果可导入到网页端关键词管理
- TODO: 是否支持AI自动拓词（生成新词）待定

### 4.5 GEO优化引导【P1】

**需求描述：**
触发GEO skill时主动引导用户，按区域逐级推进关键词优化。

**引导流程：**
1. 让用户选择优先优化哪个国家 / 省 / 市 / 区
2. 例如：优先把四川、成都的各个区县关键词文章写出来
3. 逐个发布到12个自媒体平台
4. 然后优化其他省市区的关键词

**实现方案：**
- skill 触发时检查是否有已配置的知识库+指令
- 没有则先引导配置（4.1 + 4.2）
- 有则引导选择优化区域 → 生成关键词计划 → 批量写作 → 批量发布
- 流程尽量定死，给指定格式让用户照着操作

---

### 4.6 账号管理【P2】

**需求描述：**
- 客户端授权登录后，网页端才有显示（图8）
- skill 应记住用户账号密码
- 掉线自动重新登录（根据用户提供的表格信息）

**实现方案：**
- 账号信息存储到本地配置文件（加密）
- 定期检查登录状态
- 掉线时自动重登

---

### 4.7 文章审核提醒【P2】

**需求描述：**
文章待审核时，必须用户手动通过审核才能发布（担责问题，图18）。

**实现方案：**
- 在文章写作后，skill 主动提醒"文章已进入待审核状态，请手动审核通过后才能发布"
- 不自动操作审核流程
- 提供审核指南（截图说明操作步骤）

---

### 4.7 收录查询+报表【已澄清，非独立模块】

> ⚠️ 2026-07-05 逆向验证结论：**"收录查询"不是独立功能。**
>
> 幕布原文"客户端查收录"指企业在各AI搜索引擎中的被收录情况。
> 远程API(8.138.58.181)探测结果：
> - `/api/zhushou/get_tongji` — 只返回发布统计（user_count/today_send_count/all_send_count/article_count）
