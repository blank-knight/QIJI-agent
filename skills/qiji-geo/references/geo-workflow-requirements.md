## 三大使用场景

1. **写、发（企业GEO）** — 打开GEO客户端登录后，写作文章并发布到12个自媒体平台
2. **查（收录查询）** — 用客户端查询企业GEO收录情况
3. **复盘、优化** — 半月一次：①重走AI可见度诊断→查看诊断报告 ②客户端查收录→查看数据报表

## Skill 触发条件

当用户提到关键词触发GEO skill时，skill应**主动引导**而非等待指令。流程尽量定死，给指定格式让用户照着操作，减少报错。

## 关键强约束

| 约束 | 说明 |
|------|------|
| **知识库必须真实** | 客户必须手动导入真实资料给AI，AI构建后导入奇计网页端。图片不好弄就让客户自己弄 |
| **12平台独立指令** | 每个平台必须有自己单独的指令（文章创作/标题创作/流量复刻三类）。skill根据默认指令引导用户写自己的指令 |
| **文章审核人工通过** | 文章待审核时必须用户手动通过（担责问题）。AI不能自动操作审核和发布 |
| **栏目名↔ID映射** | 官网SEO发布时，分组名必须对应官网后台栏目名。首次使用或改栏目名时，需手动给AI后台截图，让AI记住栏目和ID |
| **GEO优化按区域推进** | 让用户选优先优化哪个国家/省市区，逐级推进（如先四川成都各区县，再其他省市） |

## 网页端模块

### 知识库（图1-3）
- AI诊断页面入口：AI可见度诊断 → 诊断报告 → AI创作准备 → AI素材源力 → 关键词 → 写作标题
- 红框标注重点：**企业画像图库**和**企业知识库**（图1）
- 品牌方向多选项：经验分享/过程分享/用户痛点/用户避坑/科技前沿/行业洞察/潮流趋势等，默认全选（图2）
- 知识库预览：产品介绍/服务介绍，用通俗易懂语言描述，避免专业术语（图3）

### 指令构建（图4）
- 指令分三类标签页：文章创作 / 标题创作 / 流量复刻
- 示例指令（标题）：25-30字、加入年份2026、引起好奇心但不用夸张表述、准确反映核心内容
- 对应AI写的文章应发布到对应平台（如元宝→微信公众号）

### AI诊断平台列表（图6）
- 顶部AI平台栏：DeepSeek、兔宝、文心、纳米、Kimi、智谱等
- 诊断表单：品牌名称、行业词、是否需要优化建议
- 费用：创建AI诊断任务13元（含建议16元）

### 发布任务流程（图9-11）
- 发布选项（图9）：任务名、文章分类、任务最大发布量、每日/号发布量、文章去重（单平台去重）、AI创作声明（自动勾选）、发布平台（网易/搜狐/百家号/头条号/企鹅号/知乎/公众号/小红书）
- **网页端先启动** → 客户端检测到 → 然后发布（图10-11）
- 发布管理（图11）：开始/停止发布、自动打开浏览器开关、发布间隔（默认10秒）、统计面板（授权账号数/今日发布/累计发布/文章数）

### 个人自媒体（图8）
- 客户端授权登录后，网页端才有显示
- skill应记住用户账号密码，掉线自动重新登录（根据用户提供的表格信息）

## AI官网SEO模块

### 栏目管理（图5、14、16）
- 栏目列表含：栏目名称、编码、URL名称、模型（专题/新闻/产品/案例/招聘）、列表页模板、详情页模板、排序、状态
- 典型栏目：公司简介(aboutus)、新闻中心(article)、产品中心(product)、服务案例(case)、招聘信息(job)、联系我们(contact)

### SEO发布（图12-13、15）
- 站点管理含客户官网域名
- SEO发布任务：任务名、站点选择、发布栏目、文章分类、发布日期范围、每日发布量
- 分组名必须对应官网后台栏目名（图13-14），这样AI才能识别对应发布栏目ID
- 后台默认账号：admin / 15108232176

## 拓词工具（图21）
- 6类词库标签：前缀1(市面上/行业内/市场/目前/国内)、前缀2(口碑好的/正规的/专业的)、主词(如洗地机)、通义词(品牌/公司/工厂/厂家/供应商)、推荐词(名声/口碑)、疑问词(哪家好/哪家强/哪家靠谱)
- 组合方式：C+D、A+C+D、B+C+D、A+B+C+D、C+D+E、C+D+F等
