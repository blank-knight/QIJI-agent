# 奇计 Agent 技术方案选型对比

> 2026-06-17 | 开发者：赵文韬 + 莉雅

## 项目背景

奇计公司需要开发一款面向中小企业和个体商户的 AI Agent 产品（类似小龙虾），核心能力：

1. 集成奇计九大智能体功能
2. 通过自然语言一键操作奇计 GEO 平台（geo.heikexia.cc）
3. 第一阶段：GEO 一键操作 + 企业知识库一键搭建
4. 面向 Windows 桌面用户（小白商家）

已有资产：
- 奇计 GEO 平台（Web 后端 + Electron 客户端，功能完整）
- GEO 自动化 Skill（基于 Playwright，覆盖全部功能模块）
- 需求文档 v3（详细的网页端 + 客户端功能规格）
- 奇计 SaaS 商业体系（OEM 贴牌 → 服务商 → 企业账号）

---

## 四个候选方案

### 方案一：基于 Hermes Agent 二开

**概述：** 基于 Hermes Agent（MIT License）做二次开发，打包成奇计品牌的 Windows 桌面应用。

**Hermes 是什么：**
- Nous Research 出品的开源 AI Agent 框架
- MIT License（最宽松，完全支持商用、改名、闭源分发）
- 已有 Windows 原生桌面版（Desktop + PowerShell 安装）
- 内置 Skill 系统、浏览器自动化、记忆系统、MCP 支持
- 197k GitHub Stars，活跃维护

**架构设计：**
```
奇计 Agent 产品
├── Hermes Desktop（Windows，换品牌 + 预装 Skill）
├── Skill: GEO 一键操作（移植现有 geo-automation）
├── Skill: 企业知识库搭建（新增）
├── Skill: 九大智能体集成（后续）
├── 配置：预置奇计账号体系 + API
└── 部署：安装包 .exe，一键安装
```

| 维度 | 评分（1-5） | 说明 |
|------|------------|------|
| **商用合规** | ⭐⭐⭐⭐⭐ | MIT License，无任何限制 |
| **开发速度** | ⭐⭐⭐⭐⭐ | 我们最熟，GEO skill 可直接移植 |
| **Windows 支持** | ⭐⭐⭐⭐ | 有 Desktop + PowerShell 安装，但需美化 |
| **Skill 生态** | ⭐⭐⭐⭐ | 116 个 skill 已有，可复用 |
| **知识库** | ⭐⭐ | 没有内置，需自己加（接 MCP 向量DB） |
| **多租户/OEM** | ⭐⭐ | 没有内置，但奇计已有 web 后台管账号 |
| **品牌定制** | ⭐⭐⭐⭐ | 可换名字/Logo/配置，需改源码 |
| **社区/维护** | ⭐⭐⭐⭐⭐ | 197k Stars，Nous Research 团队维护 |
| **我们熟悉度** | ⭐⭐⭐⭐⭐ | 我跑在 Hermes 上，日常使用 |
| **小白友好度** | ⭐⭐⭐ | Desktop 偏开发者向，需套皮简化 |

**优势：**
- MIT License 最干净，商用零顾虑
- 开发速度最快（我们最熟悉）
- GEO skill 已是 Skill 格式，直接移植
- 活跃社区 + 持续维护
- 我能直接参与开发，迭代快

**劣势：**
- 没有内置知识库，需要额外开发
- 没有多租户管理（但奇计 web 后台已有）
- Desktop 面向开发者，对小白需要额外包装
- 浏览器自动化需要 Playwright（用户需额外安装依赖）

**预估开发周期：** 第一阶段 2-3 周

### 方案二：基于 Nuwax（女娲智能体）二开

**概述：** 基于 Nuwax（Apache 2.0 License）搭建完整的 Agent 平台。

**Nuwax 是什么：**
- 企业级 AI Agent 开发和运营平台
- Apache 2.0 License（支持商用，有专利限制条款）
- 完整的多仓库架构：前端(nuwax) + 后端(nuwax-backend) + PC客户端(nuwaclaw)
- 内置：知识库 + 向量检索(Milvus) + 工作流 + 插件系统 + MCP
- 支持 OpenClaw/Claude Code/Codex/KimiCli 多引擎
- 786 GitHub Stars

**架构设计：**
```
奇计 Agent 平台
├── nuwax（前端 Web，改品牌）
├── nuwax-backend（后端 API + 管理）
├── nuwaclaw（Windows PC 客户端）
├── Skill: GEO 操作（适配 Nuwax Skill 格式）
├── Skill: 知识库搭建（内置组件，配置即可）
├── 基础设施: MySQL + Redis + Milvus + Docker
└── 部署: Docker Compose / nuwaclaw 安装包
```

| 维度 | 评分（1-5） | 说明 |
|------|------------|------|
| **商用合规** | ⭐⭐⭐⭐ | Apache 2.0 支持商用，但需注意专利条款 |
| **开发速度** | ⭐⭐⭐ | 需要先学 Nuwax 源码和架构 |
| **Windows 支持** | ⭐⭐⭐⭐ | nuwaclaw 有 Win/Mac/Docker 客户端 |
| **Skill 生态** | ⭐⭐⭐ | 有 Skill 系统，但需适配格式 |
| **知识库** | ⭐⭐⭐⭐⭐ | 内置 KB + Milvus 向量检索，开箱即用 |
| **多租户/OEM** | ⭐⭐⭐⭐⭐ | 内置用户管理 + 权限 + 审计 |
| **品牌定制** | ⭐⭐⭐⭐ | 前后端分离，改品牌方便 |
| **社区/维护** | ⭐⭐⭐ | 786 Stars，团队维护但生态较小 |
| **我们熟悉度** | ⭐⭐ | 需要从头学源码 |
| **小白友好度** | ⭐⭐⭐⭐ | nuwaclaw 客户端面向终端用户 |

**优势：**
- 功能最完整，知识库/多租户/工作流开箱即用
- 企业级架构（高可用、可扩展）
- 支持多种 Agent 引擎
- nuwaclaw 客户端就是给终端用户用的

**劣势：**
- 部署复杂（MySQL + Redis + Milvus + Docker，至少 8GB 内存）
- 我们不熟悉，学习曲线陡
- Apache 2.0 专利条款需评估（虽然一般问题不大）
- 小白用户电脑上跑 Docker + Milvus 不现实
- 社区小，遇到问题靠自己
- 786 Stars vs Hermes 的 197k，长期维护有风险

**预估开发周期：** 第一阶段 4-6 周

## 推荐方案

### 推荐：方案一（Hermes）+ 方案三（OpenClaw）补充

**理由：**

1. **商用最安全**：MIT License，零顾虑
2. **开发最快**：我们最熟悉 Hermes，GEO skill 直接移植
3. **已有成功经验**：GEO skill 已在 OpenClaw 上跑通，验证了技术可行性
4. **社区最活跃**：197k Stars，长期维护有保障
5. **知识库可补**：通过 MCP 接入向量数据库（如 Chroma/Milvus），不需要自己写

**与 Nuwax 的核心差异：**
- Nuwax 的多租户/知识库/工作流确实强大，但奇计已有自己的 SaaS 后台（geo.heikexia.cc），用户管理/OEM/计费都在那边
- Agent 的角色是"操作员"，不需要做用户管理，只需要能操作平台
- 所以 Nuwax 的多租户能力对奇计来说是冗余的

**风险与缓解：**

| 风险 | 缓解方案 |
|------|---------|
| 没有内置知识库 | MCP 接入向量 DB，或自建简单 KB skill |
| Desktop 对小白不友好 | 套品牌皮 + 预装配置 + 向导式安装 |
| Playwright 依赖 | 打包到安装包里，用户无感安装 |
| 品牌定制工作量 | 改 SOUL.md + Desktop 皮肤 + 预置配置 |

### 如果后续需要完整平台能力，可追加 Nuwax

第一阶段用 Hermes 快速出 MVP，验证市场需求。如果做大做强需要多租户管理、完整知识库系统，可以追加 Nuwax 作为 SaaS 后端，前端仍用 Hermes 客户端。
