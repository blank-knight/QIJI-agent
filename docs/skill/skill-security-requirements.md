# Skill 安全保护需求文档

> **文档版本:** 1.0  
> **创建日期:** 2026-07-06  
> **状态:** 待实现

---

## 1. 背景与目标

### 1.1 背景

奇计 Claw 离线包包含多个核心 Skill（如 qiji-geo、fund-radar），这些 Skill 包含：
- 业务逻辑流程
- API 端点地址
- CSS 选择器
- 配置信息

如果这些内容被竞争对手获取，可能导致知识产权泄露。

### 1.2 核心目标

**主要目标：防止普通用户（95%）获取 Skill 源码**

次要目标（可选）：
- 防止离线包被随意复制（机器绑定）
- 防止高级用户（逆向工程师）破解

### 1.3 用户明确要求

1. **离线包与机器绑定待定**
   - 暂时不绑定机器
   - 可以复制到其他机器使用

2. **只要用户无法获取 Skill 内容就行**
   - 磁盘上的 Skill 文件必须是加密的
   - 用户直接查看时看到乱码

3. **不影响现有功能**
   - 编译流程不变
   - 安装流程不变
   - 品牌化不变
   - Git 更新不变

## 2. 要保护的内容

### 2.1 必须保护的内容

| 内容 | 路径 | 说明 |
|------|------|------|
| **Skill 文件** | `~/.hermes/skills/qiji-geo/SKILL.md` | 包含 CSS 选择器、API 端点、业务流程 |
| **Skill 文件** | `~/.hermes/skills/fund-radar/SKILL.md` | 包含基金分析逻辑 |
| **配置文件** | `~/.hermes/configs/geo-config.json` | 包含 API 密钥、凭证 |
| **可选：exe 增量更新包** | Skill Server | 包含更新差异 |

### 2.2 不需要保护的内容

| 内容 | 原因 |
|------|------|
| **hermes-agent 源码** | 已经在 Git 公开仓库 |
| **离线包 exe 本身** | 用户必须有才能运行 |
| **公开资源（图标、文档）** | 不包含敏感信息 |

## 3. 防护等级

### 3.1 阶段 1：基础防护（推荐立即实施）

**目标：防止 95% 的普通用户**

| 防护措施 | 复杂度 | 实现时间 |
|---------|-------|---------|
| Skill 文件 AES-256-GCM 加密 | ⭐⭐ 简单 | 2 小时 |
| 动态密钥派生 | ⭐⭐ 简单 | 1 小时 |
| 代码混淆 | ⭐⭐ 简单 | 30 分钟 |

**防护效果：**
- ✅ 磁盘上 Skill 文件是加密的
- ✅ 用户直接查看看到乱码
- ✅ 普通用户无法破解

**缺陷：**
- ⚠️ 逆向工程师可以通过反编译提取密钥

---

### 3.2 阶段 2：加强防护（可选）

**目标：防止 99% 的用户**

| 防护措施 | 复杂度 | 实现时间 |
|---------|-------|---------|
| 反调试（检测调试器） | ⭐⭐⭐ 中等 | 2 小时 |
| 完整性检查（exe 哈希） | ⭐⭐⭐ 中等 | 3 小时 |
| 内存保护（解密后清除） | ⭐⭐ 简单 | 1 小时 |

**防护效果：**
- ✅ 无法用调试器调试
- ✅ 无法修改 exe
- ✅ 内存中明文存活时间短

**缺陷：**
- ⚠️ 需要编译两次（完整性检查）
- ⚠️ 逆向工程师可以通过 Hook 绕过

---

### 3.3 阶段 3：终极防护（可选）

**目标：防止 99.9% 的用户**

| 防护措施 | 复杂度 | 实现时间 |
|---------|-------|---------|
| C++ 原生扩展（解密逻辑） | ⭐⭐⭐⭐ 困难 | 1 周 |
| 机器绑定（License Key） | ⭐⭐⭐ 中等 | 2 天 |
| 服务端执行（核心逻辑在服务器） | ⭐⭐⭐ 中等 | 3-5 天 |

**防护效果：**
- ✅ 难以反编译 C++ 代码
- ✅ 无法复制到其他机器
- ✅ 核心逻辑永远不出机房

**缺陷：**
- ⚠️ 需要学 C++（如果不懂）
- ⚠️ 需要 VPS 服务器（成本 30 元/月）
- ⚠️ 需要管理 Playwright（服务端执行）

## 4. 非功能性需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 解密开销 | < 1 毫秒 |
| 加密存储增加 | < 5% 文件大小 |
| 编译时间增加 | +1-2 分钟（阶段 1）<br>+15 分钟（阶段 2，编译两次） |

### 4.2 兼容性要求

| 平台 | 支持 |
|------|------|
| Windows | ✅ 完全支持 |
| macOS | ✅ 完全支持 |
| Linux | ✅ 完全支持 |

### 4.3 用户体验要求

- ✅ 用户无感知（加密/解密在内部自动处理）
- ✅ 不需要任何额外操作
- ✅ 首次启动时自动下载 Skill 文件
- ✅ 网络不可用时不影响使用（本地缓存）

### 4.4 现有功能影响

| 功能 | 是否影响 | 说明 |
|------|---------|------|
| 编译 | ❌ 不影响 | 命令不变，只是内部多了混淆步骤 |
| 安装 | ❌ 不影响 | NSIS + install.ps1 完全不变 |
| 品牌化 | ❌ 不影响 | 改动完全独立 |
| Git 更新 | ❌ 不影响 | 代码完全不变 |
| 运行性能 | ❌ 几乎无影响 | 解密开销 < 1ms |

## 7. 风险与限制

### 7.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 混淆导致代码不稳定 | 中 | 充分测试，保留未混淆版本 |
| 解密性能开销 | 低 | AES-256-GCM 很快，< 1ms |
| 编译时间增加 | 低 | 混淆在后台自动执行 |

### 7.2 安全风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 逆向工程师破解 | 高（阶段 1）<br>中（阶段 2）<br>低（阶段 3） | 分阶段实施，根据需要升级 |
| 密钥泄露 | 高 | 动态密钥派生，不在代码明文存储 |
| 服务器被攻击（服务端执行） | 中 | HTTPS + JWT + 日志审计 |

### 7.3 业务风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 用户抱怨无法复制离线包 | 中 | 明确告知这是保护措施 |
| 服务器成本（服务端执行） | 低 | 30 元/月，可忽略 |

## 8. 附录

### 8.1 术语表

| 术语 | 说明 |
|------|------|
| **AES-256-GCM** | 高级加密标准，256 位密钥，支持认证加密 |
| **代码混淆** | 通过改变代码结构和命名，使其难以理解 |
| **反调试** | 检测调试器并阻止调试 |
| **完整性检查** | 通过哈希验证 exe 是否被修改 |
| **内存保护** | 解密后立即清除明文，防止内存转储 |
| **动态密钥派生** | 运行时根据系统信息生成密钥，不静态存储 |
| **C++ 原生扩展** | 用 C++ 编写的 Node.js 模块，难以反编译 |
